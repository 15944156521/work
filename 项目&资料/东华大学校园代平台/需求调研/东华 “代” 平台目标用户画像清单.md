# **《东华 “代” 平台目标用户画像清单》**



# 东华“代”平台用户画像清单报告
## 一、报告概述
### 1.1 项目背景
东华“代”平台是面向东华大学全体学生的轻量化校园服务APP，核心定位为“解决校园高频刚需痛点的服务枢纽”，覆盖"代"业务、二手交易、新生指南三大核心模块。本次用户画像调研旨在通过明确目标用户的核心特征、痛点及需求优先级，为后续产品功能规划、流程设计提供数据支撑和方向指引，确保产品设计贴合真实校园场景需求。

### 1.2 报告目的
1.  精准定位平台核心目标用户群体，明确不同用户的差异化特征；
2.  深度挖掘各用户群体的核心痛点、核心需求及潜在期望；
3.  建立“用户-痛点-需求-功能”的对应关系，输出需求优先级矩阵；
4.  为后续产品调研、功能设计、原型开发提供核心依据。

### 1.3 调研范围与方法
1.  调研范围：东华大学大一至大四学生（含考研党），覆盖松江校区、延安路校区等多个校区；
2.  调研方法：后续将通过“问卷调研+深度访谈+个人经历”结合的方式验证画像假设，计划回收有效问卷80份，深度访谈10位典型用户。


## 二、核心用户画像详情
### 2.1 核心用户群一：大一新生
#### 2.1.1 基本属性
- 年龄：18-19岁
- 年级：大一（刚入学1-3个月）
- 校区分布：以新生集中校区为主（如松江校区）
- 经济特征：预算足够，消费意愿强（初来校园，个人生活费较为充裕）
- 信息获取渠道：班级群、辅导员通知、新生群，缺乏学长学姐私域资源

#### 2.1.2 行为特征
1.  入学初期频繁参与报到、体检、选课等校园流程，对校园环境陌生；
2.  主动搜索校园相关信息，但信息分散，筛选有效信息耗时久；（如教室地点，选课策略，兼职信息，快递地点，吃喝玩乐推荐）
3.  有购买二手教材、生活用品的需求，但担心买错版本或遭遇诈骗；
4.  倾向于群体活动，对“学长学姐推荐”的信任度极高。

#### 2.1.3 核心痛点
1.  校园流程不熟悉：报到、选课、医保办理、奖助申请等流程复杂，缺乏清晰指引，易遗漏关键步骤；
2.  校园环境陌生：找不到快递站、打印店、办事大厅等核心地点，导航软件对校园内部路线覆盖不足；
3.  信息获取困难：二手教材购买渠道单一，不清楚校园周边性价比高的餐饮、购物地点；
4.  初期适应压力大：面对陌生环境和人际关系，内心孤独，需要快速获取“生存指南”类信息。

#### 2.1.4 核心需求
1.  标准化流程指引：清晰的新生报到、选课等核心流程图文指南；
2.  精准校园导航：校园内办事地点、生活设施的详细位置及路线指引；
3.  可靠二手交易：按专业分类的二手教材专区，确保版本准确；
4.  轻量化代服务：代领校园卡、报到材料等初期高频需求（可选）。
5.  可靠信息获取源：学长学姐经验，客观信息差等。

#### 2.1.5 潜在期望
1.  学长学姐在线答疑功能；
2.  新生专属优惠（如二手交易折扣）；
3.  校园生活好物推荐（高性价比食堂、超市等）。

#### 2.1.6 产品功能匹配
- 新生指南模块：核心流程指引+校园地点导航；
- 二手交易模块：新生专属教材专区（标注专业/年级）；
- 辅助功能：新生答疑社群入口（轻量化，作为规划点）。


### 2.2 核心用户群二：大二/大三学生
#### 2.2.1 基本属性
- 年龄：19-21岁
- 年级：大二、大三
- 校区分布：跨校区上课情况普遍（如松江校区与延安路校区往返）
- 经济特征：有少量可支配收入，愿意为“节省时间”支付小额报酬，但消费较为理性。
- 学业特征：课程压力中等，课余时间碎片化，需平衡学习与生活

#### 2.2.2 行为特征
1.  频繁跨校区上课，时间紧张，常出现“取快递/外卖与上课冲突”的情况；
2.  积累了大量闲置物品（教材、电子产品、生活用品），有出售需求；
3.  信息获取渠道多元（社团群、朋友圈、校园论坛），但二手交易信息分散；
4.  对校园环境熟悉，更关注“效率提升”类服务。

#### 2.2.3 核心痛点
1.  时间冲突严重：上课、自习期间无法及时取快递、外卖，容易错过取件时间；
2.  跨校区跑腿繁琐：需往返不同校区取资料、送物品，耗时耗力，找不到合适的帮忙人员；
3.  二手交易低效：闲置物品发布在多个社群，曝光度低，成交周期长；
4.  临时需求无法满足：偶尔需要代占座、代领课堂资料等轻量化服务。

#### 2.2.4 核心需求
1.  高效代跑腿服务：代取快递/外卖、代上课/签到服务，跨校区代送资料等，流程简单、报酬透明；
2.  精准二手交易：按专业/年级给闲置物品打标签，快速匹配买家/卖家；
3.  可靠交易保障：校园内当面交易，避免诈骗；
4.  灵活接单匹配：发布需求后能快速找到合适的接单者。

#### 2.2.5 潜在期望
1.  代服务时间预估功能（明确完成时间）；
2.  二手物品估价参考；
3.  校园自提点精准标注（减少跑腿距离）。

#### 2.2.6 产品功能匹配
- 代业务模块：核心功能覆盖代取快递/外卖、跨校区代送、代占座、代领资料，代上课/打卡/签到；
- 二手交易模块：专业/年级标签筛选、校内自提点标注；
- 核心流程：需求发布-接单匹配-完成确认-报酬支付闭环。


### 2.3 核心用户群三：考研/大四学生
#### 2.3.1 基本属性
- 年龄：21-23岁
- 年级：大四（含考研党、毕业设计党）
- 校区分布：集中在图书馆、自习室周边
- 经济特征：有一定可支配收入，对“效率”和“精准度”的付费意愿高于价格敏感；
- 学业特征：时间高度紧张，专注于考研复习或毕业设计，不愿被琐事打断。

#### 2.3.2 行为特征
1.  每日长时间在图书馆、自习室备考/做设计，外出时间极少；
2.  需要大量专业复习资料、毕业设计素材，对资料精准度要求高；
3.  有闲置考研资料、专业书籍、毕业设计相关物品的出售需求；
4.  对校园服务的“便捷性”“及时性”要求极高，容忍度低。

#### 2.3.3 核心痛点
1.  琐事占用学习时间：取快递、买饭等日常琐事打断复习/设计节奏；
2.  资料获取困难：考研资料、毕业设计素材难找，正版资料价格高；
3.  占座难度大：图书馆座位紧张，需提前到场占座，耗费时间；
4.  闲置资料处理低效：考研资料、专业书籍针对性强，难以快速转手给有需求的低年级学生。

#### 2.3.4 核心需求
1.  全流程代服务：代买饭、代取快递、代占座，覆盖日常琐事，代送资料，代打印；
2.  精准资料交易：考研/毕业设计资料专区，确保资料版本准确、针对性强；
3.  高效资料互换：支持以物换物，降低资料获取成本；
4.  快速交易匹配：闲置资料能快速对接低年级有需求的学生。

#### 2.3.5 潜在期望
1.  资料真伪验证功能；
2.  代服务定时配送（如固定三餐配送时间）；
3.  考研专属占座提醒（避免座位被占）。

#### 2.3.6 产品功能匹配
- 代业务模块：代买饭、代取快递、代占座（重点优化定时服务）；
- 二手交易模块：考研资料专区、毕业设计素材专区；
- 增值功能：资料互换专区（以物换物）。


## 三、用户需求优先级矩阵
### 3.1 优先级定义标准
- P0（必须做）：覆盖核心用户高频刚需痛点，直接决定产品核心价值；
- P1（应该做）：覆盖核心用户中频需求，提升产品竞争力；
- P2（可以做）：覆盖补充用户需求或潜在期望，丰富产品功能；
- P3（暂不做）：需求频次低、覆盖用户少，或实现成本高，后续迭代再考虑。

### 3.2 需求优先级明细
| 需求类别     | 具体需求内容               | 对应用户群           | 优先级 | 核心依据                                 |
| ------------ | -------------------------- | -------------------- | ------ | ---------------------------------------- |
| 代业务需求   | 代取快递/外卖              | 大二/大三、考研/大四 | P0     | 覆盖核心用户高频刚需，痛点最强烈         |
| 代业务需求   | 跨校区代送资料             | 大二/大三            | P0     | 跨校区学生核心痛点，需求频次高           |
| 代业务需求   | 代占座、代领课堂资料       | 大二/大三、考研/大四 | P1     | 中频需求，提升用户效率                   |
| 代业务需求   | 代买饭、代领校园卡         | 考研/大四、大一      | P1     | 覆盖精准用户需求，提升产品覆盖面         |
| 二手交易需求 | 专业/年级标签筛选          | 全用户群             | P0     | 解决信息分散问题，提升交易效率           |
| 二手交易需求 | 校内自提点标注             | 大二/大三、考研/大四 | P0     | 降低交易成本，提升用户体验               |
| 二手交易需求 | 新生教材专区、考研资料专区 | 大一、考研/大四      | P1     | 精准匹配需求，提升产品针对性             |
| 二手交易需求 | 资料互换（以物换物）       | 考研/大四            | P2     | 潜在需求，覆盖用户少                     |
| 新生指南需求 | 核心流程指引+校园导航      | 大一                 | P0     | 新生初期高频刚需，直接解决适应压力       |
| 新生指南需求 | 学长学姐答疑               | 大一                 | P2     | 潜在期望，可通过社群轻量化实现，暂不上线 |
| 增值需求     | 代服务时间预估、定时配送   | 考研/大四            | P2     | 提升体验，但非核心需求                   |
| 增值需求     | 资料真伪验证、物品估价     | 全用户群             | P3     | 实现成本高，需求频次低                   |

## 四、总结
### 4.1 核心结论
1. 平台核心用户为大二/大三学生，核心价值是“通过代业务+二手交易解决时间冲突、效率低下的痛点”；

2. 大一新生和考研/大四学生为补充用户，分别通过“新生指南”和“精准资料交易”提升平台覆盖面；

3. 产品设计需优先聚焦P0级需求（代取快递/外卖、跨校区代送、专业标签筛选、新生流程指引），确保核心价值落地；

   **后续工作**

1.  需求调研验证：后续通过问卷调研验证各用户群痛点的真实性和需求频次，重点确认P0级需求的用户占比；
2.  功能规划依据：按“P0→P1→P2”的优先级规划功能模块，核心模块（代业务、二手交易）优先落地；
3.  原型设计重点：核心流程（需求发布-接单匹配-交易完成）需简洁直观，符合学生使用习惯；
4.  差异化设计：针对不同用户群的需求特点，在模块内设置专属专区（如新生教材专区、考研资料专区），提升精准度。
