# 东华 “代” 平台用户数据分析报告

## 一、项目背景与报告目标

### 1.1 项目背景

东华 “代” 平台是面向东华大学学生的轻量化校园服务产品，聚焦 “代业务（代取快递 / 外卖等）、二手交易（教材 / 资料等）、新生指南（流程 / 导航等）” 三大核心模块，旨在解决校园高频刚需痛点。本次调研基于 80 份有效问卷数据，量化用户需求特征，为产品功能设计提供数据支撑。

### 1.2 报告目标

1. 验证核心用户（大二 / 大三学生）的痛点频次与需求强度，明确功能优先级；
2. 量化性别差异（付费意愿、沟通成本、信誉表现），支撑差异化功能设计；
3. 提炼二手交易、新生指南模块的核心诉求，优化模块功能框架；
4. 输出 “数据结论→产品设计” 的映射关系，直接指导原型开发。

## 三、核心维度数据分析

### 3.1 性别差异分析（产品差异化设计关键依据）

| 分析维度           | 女生数据表现                              | 男生数据表现                              | 核心差异结论                                      |
| ------------------ | ----------------------------------------- | ----------------------------------------- | ------------------------------------------------- |
| 付费意愿（代业务） | 选择 7-10 元及以上占比 65%（26/40）       | 选择 0-3 元占比 50%（20/40）              | 女生付费能力显著高于男生，溢价接受度高            |
| 沟通清晰率         | “每次 / 大部分能说清需求” 占 85%（34/40） | “每次 / 大部分能说清需求” 占 55%（22/40） | 男生沟通成本高，易遗漏关键信息（如地址 / 取件码） |
| 二手交易纠纷率     | 1-2 次及以上纠纷占 20%（8/40）            | 1-2 次及以上纠纷占 45%（18/40）           | 男生交易信誉风险高于女生，需加强保障              |
| 加急服务接受度     | 愿意多付 3-5 元占 40%（16/40）            | 愿意多付 3-5 元占 10%（4/40）             | 女生对 “时效性” 需求更强，可设计增值服务          |

### 3.2 代业务需求分析（核心模块 P0 级功能确认）

#### 3.2.1 需求频次与优先级

| 代业务类型        | 每周 3-5 次及以上占比 | 核心用户群体          | 需求优先级 | 依据                               |
| ----------------- | --------------------- | --------------------- | ---------- | ---------------------------------- |
| 代取快递 / 外卖   | 45%（36/80）          | 大二 / 大三女生       | P0         | 覆盖最广、频次最高，女生付费意愿强 |
| 跨校区代送资料    | 37.5%（30/80）        | 大二 / 大三跨校区学生 | P0         | 跨校区场景刚需，解决时间成本问题   |
| 代买饭 / 代打印   | 25%（20/80）          | 考研党 / 大四学生     | P1         | 精准用户需求，频次低于前两类       |
| 代占座 / 代领资料 | 20%（16/80）          | 考研党 / 大二学生     | P1         | 场景特定，非全民刚需               |

#### 3.2.2 核心痛点（基于 “代服务最担心问题” 数据）

1. 对方不靠谱（违约 / 丢失物品）：占比 42.5%（34/80），全用户群共性痛点；
2. 沟通不顺畅：占比 35%（28/80），男生占比超 60%，性别差异显著；
3. 价格不透明：占比 27.5%（22/80），跨校区代送场景最突出。

### 3.3 二手交易需求分析（模块功能优化方向）

#### 3.3.1 核心痛点（基于 “出售闲置困扰” 数据）

| 痛点类型             | 占比         | 典型用户群体          | 解决优先级 |
| -------------------- | ------------ | --------------------- | ---------- |
| 信息没人看（曝光低） | 40%（32/80） | 大二 / 大三卖教材用户 | P0         |
| 买家沟通反复         | 35%（28/80） | 全用户群              | P0         |
| 自提麻烦             | 30%（24/80） | 跨校区交易用户        | P1         |
| 担心交易诈骗         | 25%（20/80） | 大一新生 / 男生       | P0         |

#### 3.3.2 用户核心诉求（基于 “购买二手看重” 数据）

1. 专业 / 年级标签筛选：占比 65%（52/80），解决 “找错教材 / 资料” 问题；
2. 卖家信誉可查：占比 60%（48/80），男生用户需求强度最高；
3. 物品实拍图 + 细节描述：占比 55%（44/80），减少 “货不对板” 纠纷。

### 3.4 新生指南需求分析（精准匹配新生痛点）

基于大一用户（16 人）数据，核心需求排序：

1. 报到 / 选课 / 医保流程指引：占比 81.25%（13/16），入学初期高频刚需；
2. 校园内地点导航（快递站 / 打印店）：占比 75%（12/16），解决 “找路难” 问题；
3. 二手教材购买渠道：占比 62.5%（10/16），结合预算有限的特征；
4. 校园周边推荐：占比 50%（8/16），非核心但提升体验的补充需求。

## 四、核心结论与产品设计映射

| 数据分析结论                                                 | 对应产品功能设计                                             | 模块归属 | 设计价值                               |
| ------------------------------------------------------------ | ------------------------------------------------------------ | -------- | -------------------------------------- |
| 1. 女生高付费 + 加急需求（65% 愿付 7-10 元，40% 愿多付 3-5 元加急） | 代业务模块：增设 “加急溢价” 功能（1-2 元 / 3-5 元两档），默认优先向女生用户推荐 | 代业务   | 满足高价值用户时效性需求，提升平台收益 |
| 2. 男生沟通成本高（55% 沟通不清晰，易漏地址）                | 代业务模块：需求发布页增加 “标准化模板”（强制填写校区 + 楼栋 + 取件码 / 时间） | 代业务   | 降低沟通成本，减少履约延误             |
| 3. 男生交易纠纷率高（45% 有纠纷）+ 全用户需信誉保障          | 二手交易模块：① 卖家标签体系（女生默认 “优质卖家” 候选，男生需 3 次无纠纷解锁）② 交易担保功能（先付款至平台，确认无误后打款） | 二手交易 | 降低信任风险，提升交易转化率           |
| 4. 代取快递 / 跨校区代送为 P0 需求（频次最高）               | 代业务模块：首页优先展示 “代取快递”“跨校区代送” 入口，简化操作流程 | 代业务   | 聚焦核心需求，提升用户使用效率         |
| 5. 二手交易需专业 / 年级标签（65% 用户看重）                 | 二手交易模块：发布商品时强制选择 “专业 + 年级” 标签，搜索页支持双维度筛选 | 二手交易 | 精准匹配供需，解决 “信息没人看” 痛点   |
| 6. 新生需流程指引 + 校园导航（81.25%/75% 需求）              | 新生指南模块：① 按 “报到 / 选课 / 医保” 分场景分步指引② 嵌入校园内部精准导航（标快递站 / 打印店） | 新生指南 | 解决新生核心痛点，提升模块实用性       |

## 五、后续产品推进建议

### 5.1 功能开发优先级

1. 优先落地 P0 级功能：代取快递 / 跨校区代送流程、二手交易专业标签 + 交易担保、新生流程指引 + 导航，代上课；
2. 次优先落地差异化功能：女生加急溢价、男生需求模板；
3. 后续迭代功能：代买饭 / 代打印、二手资料互换。
